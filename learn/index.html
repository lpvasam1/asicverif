A note to the reader

How to use this manual.

This manual is organized as a book in three parts — Foundations, Construction, and Mastery — each a cluster of chapters that together form a complete study of the SystemVerilog Hardware Verification Language. The first part attends to the language's grammar: its data types, its procedural statements, its connection to the Device Under Test. The second part turns the grammar into architecture: interfaces, clocking blocks, object-oriented testbenches, and constrained-random stimulus. The third part is where the language repays the study — threads, polymorphism, coverage, the DPI, and the putting-together of a complete environment.

Each part closes with a proficiency exam of twenty multiple-choice questions, auto-graded the moment you submit, with an explanation beneath every item. The exams are intended to be retaken. A student who can answer all sixty with confidence is ready for UVM.

A final word on the code plates scattered throughout. SystemVerilog is a language of small programs, not short sentences — the smallest interesting example is usually a dozen lines, not two. Read the plates slowly. Every keyword earns its place.

Plate I

"A constrained-random transaction" — the smallest program that exercises roughly half the language.

transaction.sv 30 lines · IEEE 1800-2009
// A constrained-random bus transaction class Transaction; rand bit [31:0] src, dst, data[]; rand bit [7:0] len; randc bit [1:0] kind; // cycles through values static int count = 0; // class-level counter int id; constraint c_len { len inside {[1:32]}; } constraint c_data { data.size() == len; } constraint c_weight { kind dist {0 := 5, 1 := 3, 2 := 1}; } function new(); id = count++; // every object gets a unique id endfunction covergroup cg; cp_len : coverpoint len { bins b[8] = {[1:32]}; } cp_kind : coverpoint kind; cross_lk: cross cp_len, cp_kind; endgroup endclass
